# User Flow

1. 使用者打開「地點」頁籤，看到所有站點列表：名稱、目前狀態 badge、來源、最後確認時間
2. 點一個有待確認建議的站點，畫面顯示建議內容、建議者角色、建議的驗證狀態，並清楚標示「這是建議，不是已生效狀態」
3. 使用者依目前站點狀態按下對應動作（採用建議 / 標記需複查 / 駁回建議），畫面立即顯示結果與簡短原因；沒有待確認建議的站點則顯示「目前沒有待確認的建議」

## What can go wrong?

- 建議與目前狀態互相矛盾，但使用者沒注意標示文字，誤以為建議已經生效 → 用顏色與明確文案（「建議」字樣＋警示色）強化區別
- 使用者對已確認（verified）站點按下「標記需複查」，可能被誤解成「已經證實關閉/開放」→ 動作完成後的文案明確寫出「尚未變更為建議狀態」
- 同一站點若未來出現多筆建議，目前設計只處理其中一筆，容易讓人以為已經處理完所有建議（已列在 `docs/spec.md` 的 out of scope，留給下一位接手者）
